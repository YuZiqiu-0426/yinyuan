# 当前实现说明

## 1. 文档目的

本文档记录 `y2m-rs` 当前已经完成的实现、已经验证的能力，以及尚未完成的部分，便于后续继续开发、联调和补测试。

当前主代码目录：

```text
y2m-rs/
```

该目录是一个独立的 Rust workspace。

## 2. 当前项目结构

当前已经落地的项目结构如下：

```text
y2m-rs/
  Cargo.toml
  src/
    main.rs
  crates/
    common/
    server/
    client-core/
  tests/
    support/
    text_e2e.rs
    json_e2e.rs
    command_e2e.rs
    file_transfer_v3.rs
```

其中：

1. `crates/common`：协议模型、公共枚举、协议常量、二进制分片头、跨平台命令辅助方法
2. `crates/server`：服务端 `init / router / ws / session / transfer` 主链路
3. `crates/client-core`：客户端配置、连接、协议解析、运行时、插件注册、命令包构造
4. `src/main.rs`：`y2m` CLI 入口，负责 `init / run / send / chat`
5. `tests/`：当前 workspace 的端到端测试与测试辅助模块

## 3.5 文件传输当前状态

当前文件传输链路已经可以跑通：

1. 发送端支持 `send file` 和 `chat /file`
2. 接收端收到 `file_offer` 后不再自动接受
3. 接收端支持：
   - `/files` 查看待确认和接收中的文件
   - `/accept <fileId>` 接收
   - `/reject <fileId>` 拒绝
   - `/abort <fileId>` 取消待确认或接收中的文件
4. 发送端支持：
   - `send file` 的阻塞式发送
   - `chat /file` 的后台发送
   - `chat /abort <fileId>` 取消外发文件
5. 接收文件时会：
   - 自动选择保存路径
   - 避免重名覆盖
   - 按配置中的 `download_dir` 或默认 `downloads/` 保存
   - 校验 `fileSize`
   - 校验 `sha256`
   - 校验通过后落盘
   - 返回 `ack`
6. 当前发送端和接收端都已经带有进度/阶段日志
7. `file_abort` 当前已完成：
   - 协议包构造
   - 服务端转发和传输状态清理
   - 发送侧收到 `file_abort` 后停止继续发送
   - 接收侧收到 `file_abort` 后清理本地接收状态
8. 已补的错误处理：
   - 未 `accept` 前发送 binary chunk 会被拒绝，并向发送方返回错误包
   - 非法 binary chunk 会向发送方返回 `invalid_message`
   - 缺失传输状态时的 `file_complete` 不会继续转发
   - `file_offer` 超过服务端 `max_file_size / max_chunk_size` 会直接被拒绝
   - `file_complete` 收到 `ack=rejected` 后会清理服务端传输状态
9. 当前最小重连一致性处理：
   - `run/chat` 断线后会自动重新 `init`
   - 重连前会清理待确认/接收中/外发中的文件状态
   - 重连成功后会本地回放未完成文件已失败，并提示“等待对方在线后重新发送”
10. 本地文件状态结构当前已进一步收敛：
   - `ConsoleState` 底层已改为单个 `LocalFileStore` 承载待确认 / 接收中 / 外发中三类状态
   - `LocalFileStore` 内部已进一步收敛为 `fileId -> LocalFileTransfer` 单映射，每个文件只对应一个本地状态对象
   - 原有状态读写 helper 已切到统一存储入口，减少多把锁分散访问
   - `LocalFileTransfer` 已开始采用共享字段 + `LocalFileState` 显式阶段表达，本地外发阶段不再额外维护独立枚举
   - `LocalFileTransfer` 现已内聚常见状态谓词与阶段迁移入口，helper 不再分散维护外发阶段判断
   - `move_to_incoming()` / `transition_to()` 已从布尔返回值改为结构化迁移错误，非法迁移原因可以被单独建模

   - 真实 `y2m` 进程下的 `command -> command_result` 超时路径
   - 真实 `y2m` 进程下的 `command -> command_result` 命令失败路径
   - 真实 `run` 进程在服务端重启后的自动重连

   - 真实 `chat` 进程在外发文件等待对方确认时断线重连，会在重连成功后回放“发送失败，需要重发”的本地提示
   - 真实 `chat` 进程在接收中文件时断线重连，会在重连成功后回放“传输失败，需要重发”的本地提示
   - 真实 `chat` 进程下的 `/file -> /files -> /accept`
   - 真实 `chat` 进程下的 `/reject`
   - 真实 `chat` 进程下的 `/abort`
   - 真实 `chat` 进程下的文件校验失败、不落盘、`/files` 清空与服务端状态清理
   - 真实 `chat` 进程下的 `fileSize` 不匹配校验失败、不落盘、`/files` 清空与服务端状态清理

   - 真实 `chat` 进程下的重复 chunk 会立刻失败、本地清理并向发送方回 `file_abort`
   - 真实 `chat` 进程下的 `totalChunks` 不一致会立刻失败、本地清理并向发送方回 `file_abort`
   - 真实 `chat` 进程下的多 chunk 缺片在 `file_complete` 时会被拒绝、本地清理且后续 chunk 会被服务端拒绝
7. `src/main.rs` 单元测试
   - 非法 `move_to_incoming()` 会返回显式 `UnexpectedState`
   - 非法 `transition_to()` 会返回显式 `InvalidTransition`
   - 合法 `move_to_incoming()` 会重置进度并切到 `Receiving`

## 4. 当前未完成部分

以下内容当前仍未完成，或者只完成了最小可用版本：

1. 完整客户端断线重连语义
   - 文件传输进行中断线后的恢复/取消策略已开始收敛到统一失败原因模型，仍可继续细化到更正式的状态机
   - 重连期间本地暂存态与服务端状态的一致性语义已覆盖待确认/接收中/外发等待确认三类典型场景
   - `/files` 已改为基于统一本地文件条目快照输出，并补本地状态标签展示（待确认 / 接收中 / 等待对方接受 / 发送中 / 等待对方确认）
   - 重连清理回放与 `/files` 展示已开始共用同一层本地文件条目视角
   - 本地文件状态底层已收口为单个 `LocalFileStore` + `LocalFileTransfer` 单对象存储，并已改为共享字段 + `LocalFileState` 显式阶段表达；常见迁移判断与非法迁移错误已开始内聚到 `LocalFileTransfer`，但仍可继续演进为更正式的统一迁移表

   - 仍可继续补更细的 chunk 边界场景（如更多 header 字段异常组合）
4. 独立业务 plugin crate
   - 当前仍以 `ConsolePlugin` 为主
   - 还没有拆成独立的 `message/json/command/file` 插件 crate
5. 服务端更完整的运维能力
   - 更细的日志字段
   - 监控指标
   - 更完善的异常清理
6. 更完整的配置体系

   - 超时策略
