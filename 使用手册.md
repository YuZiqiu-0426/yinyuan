# y2m-rs 使用手册

## 1. 文档目的

本文档面向最终使用者，说明 `y2m` 的基础使用方法，包括初始化、消息发送、命令执行、文件传输和 `chat` 交互命令。

## 2. 角色说明

系统包含两个角色：

1. `y2m-server`：消息转发服务端
2. `y2m`：客户端命令行工具

本文重点介绍客户端 `y2m` 的使用。

## 3. 基本概念

### 3.1 server-url

客户端连接的 WebSocket 地址，例如：

```text
ws://127.0.0.1:8080/ws
```

### 3.2 group

分组名。广播默认发往当前分组。

### 3.3 client

客户端名。单播时通过客户端名指定目标。

### 3.4 run 与 chat

1. `run`：常驻接收模式，适合后台运行或控制台观察
2. `chat`：交互式会话模式，适合人工聊天和文件操作

## 4. 首次使用

首次建议先初始化配置：

```powershell
y2m init `
  --server-url ws://127.0.0.1:8080/ws `
  --group demo `
  --client alice `
  --heartbeat-interval 10 `
  --download-dir .\downloads
```

常用参数：

1. `--server-url`：服务端地址
2. `--group`：默认分组
3. `--client`：当前客户端名
4. `--token`：预留认证字段
5. `--heartbeat-interval`：心跳间隔覆盖
6. `--download-dir`：接收文件的保存目录

## 5. 配置文件位置

默认配置文件路径：

Windows：

```text
%APPDATA%\y2m\config.json
```

Linux/macOS：

```text
~/.config/y2m/config.json
```

如果需要指定其他配置文件，可在相关命令中加：

```powershell
--config <path>
```

## 6. 启动方式

### 6.1 run 模式

启动接收控制台：

```powershell
y2m run
```

开启自动重连：

```powershell
y2m run --reconnect-interval-sec 5
```

适合场景：

1. 作为消息接收端长期在线
2. 接收文本、JSON、命令和文件请求
3. 接收 `command` 并自动执行后回传 `command_result`

### 6.2 chat 模式

进入交互会话：

```powershell
y2m chat
```

指定默认目标用户：

```powershell
y2m chat --to bob
```

指定默认目标分组：

```powershell
y2m chat --group demo
```

带自动重连：

```powershell
y2m chat --reconnect-interval-sec 5
```

## 7. 单次发送命令

### 7.1 发送文本

发给指定用户：

```powershell
y2m send text --to bob "hello"
```

发给指定分组：

```powershell
y2m send text --group demo "hello all"
```

说明：

1. 指定 `--to` 时按单播发送
2. 只指定 `--group` 时按分组广播发送

### 7.2 发送 JSON

```powershell
y2m send json --to bob "{\"type\":\"ping\",\"value\":1}"
```

或：

```powershell
y2m send json --group demo "{\"event\":\"notice\"}"
```

说明：

1. 传入内容必须是合法 JSON 字符串
2. 服务端会按 JSON 事件路由

### 7.3 发送命令

```powershell
y2m send command --to bob "dir" --timeout 30
```

Linux/macOS 示例：

```bash
y2m send command --to bob "ls -la" --timeout 30
```

说明：

1. `command` 当前仅支持单播
2. `--timeout` 为等待结果超时秒数
3. 接收端会执行命令并返回 `command_result`

### 7.4 发送文件

```powershell
y2m send file --to bob ".\demo.txt" --timeout 30
```

说明：

1. 文件发送当前仅支持单播
2. 接收方需要明确接受
3. 发送端会经历：`file_offer -> file_accept -> 分片发送 -> file_complete -> ack`

## 8. chat 交互命令

进入 `chat` 后，可使用以下命令：

```text
/to <client>      切换目标用户
/to               清空目标用户并恢复广播
/group <group>    切换目标分组
/group            清空目标分组并恢复默认分组
/json <json>      发送 JSON 消息
/command <cmd>    发送命令请求
/file <path>      发送文件
/files            查看本地文件状态
/accept <fileId>  接收待确认文件
/reject <fileId>  拒绝待确认文件
/abort <fileId>   取消发送或接收中的文件
/status           查看当前会话
/help             查看帮助
/exit             退出会话
```

### 8.1 普通文本输入

在 `chat` 模式下直接输入文本，会按当前会话目标发送文本消息。

例如：

```text
hello
```

### 8.2 切换当前目标

切换到用户 `bob`：

```text
/to bob
```

切换当前分组：

```text
/group test
```

恢复默认广播：

```text
/to
/group
```

## 9. 文件传输使用方式

### 9.1 接收方流程

接收方在 `run` 或 `chat` 中收到文件请求后，会看到：

1. 文件 ID
2. 来源用户
3. 文件名
4. 文件大小
5. 默认保存路径

可执行：

```text
/files
/accept <fileId>
/reject <fileId>
```

### 9.2 发送方流程

发送方在 `chat` 或 `send file` 中发起发送后，会看到：

1. 已发送文件申请
2. 对方是否接受
3. 分片发送进度
4. 是否完成校验确认

### 9.3 中止文件传输

无论是待确认、接收中还是发送中，都可执行：

```text
/abort <fileId>
```

### 9.4 查看本地文件状态

```text
/files
```

当前会显示类似状态：

1. `待确认`
2. `接收中`
3. `等待对方接受`
4. `发送中`
5. `等待对方确认`

## 10. 自动重连说明

`run` 和 `chat` 支持最小自动重连：

```powershell
y2m run --reconnect-interval-sec 5
y2m chat --reconnect-interval-sec 5
```

当前行为：

1. 断线后会自动重新连接并重新 `init`
2. 断线时未完成的文件传输会判定失败
3. 重连成功后会在本地回放失败提示

当前限制：

1. 文件传输断线后不会断点续传
2. 需要等待对方在线后重新发送

## 11. 命令执行说明

接收端在收到 `command` 后，会在本地执行命令并回传：

1. `exitCode`
2. `stdout`
3. `stderr`
4. `durationMs`

当前约定：

1. Windows 默认使用 `cmd /C`
2. Linux/Unix 默认使用 `sh -c`

## 12. 常见使用示例

### 12.1 两个客户端互发文本

客户端 A：

```powershell
y2m init --server-url ws://127.0.0.1:8080/ws --group demo --client alice
y2m chat
```

客户端 B：

```powershell
y2m init --server-url ws://127.0.0.1:8080/ws --group demo --client bob
y2m run
```

然后客户端 A 执行：

```text
/to bob
hello
```

### 12.2 发送 JSON

```text
/to bob
/json {"type":"notice","message":"hello"}
```

### 12.3 发送命令

```text
/to bob
/command dir
```

### 12.4 发送文件

```text
/to bob
/file .\demo.txt
```

接收方可执行：

```text
/files
/accept <fileId>
```

## 13. 常见问题

### 13.1 为什么发送文件后没有立刻开始传输

因为当前文件发送需要接收方显式接受。

### 13.2 为什么断线后文件没继续传

当前实现是最小一致性策略：断线按失败处理，需要重新发送。

### 13.3 `command` 为什么建议只对可信目标使用

因为它会在接收端本地执行命令，存在明显的安全风险。

### 13.4 服务端地址怎么写

当前客户端默认地址是：

```text
ws://127.0.0.1:8080/ws
```

如果服务端部署在其他机器，请替换成对应 IP 或域名。

## 14. 当前限制

当前尚未完全补齐的基础能力包括：

1. 文件断点续传或中断恢复
2. 更完整的自动接受策略
3. 更完整的文件超时和 chunk 大小配置
4. 更细粒度的命令等待与交互策略
