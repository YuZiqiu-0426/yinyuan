# 错误码与异常语义规范 v1

## 1. 返回结构

```json
{
  "code": "AUTH_PERMISSION_DENIED",
  "message": "permission denied",
  "requestId": "req_xxx"
}
```

## 2. 错误码映射

- `AUTH_INVALID_CREDENTIALS` -> 401
- `AUTH_TOKEN_EXPIRED` -> 401
- `AUTH_REFRESH_EXPIRED` -> 401
- `AUTH_CSRF_INVALID` -> 403
- `AUTH_IP_MISMATCH` -> 403
- `AUTH_DEVICE_UNTRUSTED` -> 403
- `AUTH_CLI_REVIEW_PENDING` -> 403
- `AUTH_CLI_REVIEW_REJECTED` -> 403
- `AUTH_PERMISSION_DENIED` -> 403
- `AUTH_SESSION_READONLY` -> 403
- `AUTH_SESSION_REVOKED` -> 401
- `AUTH_GROUP_SCOPE_DENIED` -> 403
- `AUTH_RISK_REVOKED` -> 401
- `AUTH_SESSION_STATE_INVALID` -> 409

## 3. 可重试策略

- 可重试：网络错误、5xx
- 条件可重试：`AUTH_TOKEN_EXPIRED`（刷新后重试）
- 不可重试：权限、范围越权、设备不可信、审核拒绝

## 4. 前端提示原则

- 不暴露内部安全细节
- 登录失败统一提示“账号或凭据错误”
- 权限拒绝提示“当前会话无操作权限”

## 5. 观测字段

- 每个错误响应必须包含 `requestId`
- 服务端日志需记录 `requestId + userId + ip + code`
