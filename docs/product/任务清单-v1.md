## 模块二：auth-service（Rust + Axum）

> 新 crate，建议放 `auth-service/`（独立 workspace 或加入现有 workspace 均可）。

### 2.1 基础脚手架

| # | 任务 | 状态 |
|---|------|------|
| AS-01 | 初始化 `auth-service` crate：Axum + SQLx + Redis + JWT + Argon2id + tracing | ⬜ |
| AS-02 | 配置分层：`dotenvy` 读取所有必要环境变量（见 `配置与密钥管理规范-v1.md` §2） | ⬜ |
| AS-03 | 建 `domain` 模块：User / Role / Permission / Device / Session 核心实体与业务规则 | ⬜ |
| AS-04 | 建 `infra` 模块：Postgres 连接池（SQLx）、Redis 连接、JWT 签发/校验（RS256）、Argon2id 密码哈希 | ⬜ |
| AS-05 | 建 `audit` 模块：审计写入入口（异步，追加写，不可改） | ⬜ |

### 2.2 Web 认证

| # | 任务 | 状态 |
|---|------|------|
| AS-06 | `POST /api/v1/auth/web/login`：账号密码校验、签发 Access Token + Refresh Token（HttpOnly Cookie） | ⬜ |
| AS-07 | `POST /api/v1/auth/web/refresh`：Refresh Token 轮换、CSRF Token 校验（`X-CSRF-Token` + Origin/Referer）、重放检测 → `revoked` | ⬜ |
| AS-08 | `POST /api/v1/auth/web/logout`：吊销当前会话 | ⬜ |
| AS-09 | 登录限流（用户名 + IP 维度，Redis 计数窗口） | ⬜ |

### 2.3 CLI 认证与审核流

| # | 任务 | 状态 |
|---|------|------|
| AS-10 | `POST /api/v1/auth/cli/login`：设备指纹计算、`deviceSignature` 验签、首次登录写 `cli_login_reviews`（`pending`） | ⬜ |
| AS-11 | 复审后登录：`cli_login_reviews.status=approved` 时签发 CLI Access Token | ⬜ |
| AS-12 | 每次会话设备因子校验：`mac + osLoginUser` 签名校验 + 服务端源 IP（不信任客户端上报） | ⬜ |
| AS-13 | `POST /api/v1/auth/cli/logout`：吊销 CLI 会话 | ⬜ |

### 2.4 鉴权中间件（供 chat-service 复用）

| # | 任务 | 状态 |
|---|------|------|
| AS-14 | `verify_access_token`：JWT 签名 + 标准字段校验 | ⬜ |
| AS-15 | `verify_ip_binding`：来源 IP 与 token 绑定 IP 比对；支持 CIDR 白名单策略 | ⬜ |
| AS-16 | `load_permission_snapshot`：从 token claims 或 Redis 缓存加载权限快照 | ⬜ |
| AS-17 | `authorize(action)`：基于动作映射表（见 `详细设计` §9.1）做原子权限判定 | ⬜ |
| AS-18 | 会话状态校验：每次操作前检查 `session_state`（`revoked` 断链，`suspended_readonly` 禁发送） | ⬜ |
| AS-19 | 将 AS-14~18 打包为独立 library crate，供 chat-service 引入 | ⬜ |

### 2.5 身份查询

| # | 任务 | 状态 |
|---|------|------|
| AS-20 | `GET /api/v1/auth/me`：返回用户、角色、权限、组、会话状态 | ⬜ |
| AS-21 | `POST /api/v1/auth/introspect`：服务间 token 校验接口 | ⬜ |

### 2.6 Admin API

| # | 任务 | 状态 |
|---|------|------|
| AS-22 | `POST /admin/users`：创建用户，`super_admin` 跨组，`group_admin` 仅本组（否则返回 `AUTH_GROUP_SCOPE_DENIED`） | ⬜ |
| AS-23 | `PATCH /admin/users/{id}/status`：启用/禁用用户（高危，写审计） | ⬜ |
| AS-24 | `POST /admin/groups`、`POST /admin/user-group-roles`：组与角色分配 | ⬜ |

| AS-26 | `GET /admin/devices`、`PATCH /admin/devices/{id}/status`：设备封禁 | ⬜ |
| AS-27 | `POST /admin/cli-reviews/{reviewId}/approve` 与 `/reject`：CLI 首登审核 | ⬜ |
| AS-28 | `GET /admin/audit-logs`：按用户、IP、时间区间检索（支持导出） | ⬜ |
