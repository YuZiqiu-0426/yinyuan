## 3. 已实现（✅）

### 3.1 协议与公共库（`y2m-common`）

- ✅ v3 包模型、`EventType` / 各类载荷、二进制分片头
- ✅ 跨平台 shell 辅助（协议层约定；Windows 上 **执行** `command` 时 CLI 使用 `pwsh`，见 `当前实现说明` §3.7）

### 3.2 服务端（`y2m-server`）

- ✅ 会话、`init`、心跳与超时清理、WebSocket 主链路
- ✅ 路由：**`text` / `json` / `command` / `command_result` / `file_*`** 均支持**单播与组内广播**（有明确目标则单播）
- ✅ 文件 v3：`TransferRegistry`、多接收端腿、`file_accept` 后 chunk 中转、`file_complete` / `ack` / 清理
- ✅ 日志落盘（`tracing-appender`）、默认监听与 `Y2M_SERVER_ADDR`

### 3.3 客户端核心（`y2m-client-core`）

- ✅ 配置、连接、运行时、插件分发、各类 `build_*_packet`
- ✅ **发送端信封**：出站事件 `metadata` 统一附带 `senderIp` / `senderMac` / `senderUser` / `senderOs`（Windows `getmac`、Linux sysfs、macOS `/sbin/ifconfig`）

### 3.4 CLI（`y2m`）

- ✅ `init` / `run` / `send` / `chat` 全链路；`chat` 使用 **reedline** + `ExternalPrinter`
- ✅ `send text|json|command|file`：无 `--to` 时可走**组播**（与路由一致；打印目标可为 `*`）
- ✅ 文件：`send file` 与 `chat /file`；**配置中显式 `download_dir` 时自动 `file_accept`**，否则需 `/accept`
- ✅ **`command_result` 控制台**：按 `request_id` 防抖聚合、按用户名字典序打印；`send command` 结束时立即 flush
- ✅ 对端展示发送端终端摘要（文本/JSON/命令/文件 offer/命令结果块）
- ✅ 重连：清理文件态 + 回放失败提示；`command_result` 聚合缓冲随重连重置

### 3.5 测试与工程

- ✅ 进程内 e2e：`text` / `json` / `command` / `file_transfer_v3` / `heartbeat_timeout` 等
- ✅ 辅助断言：`metadata` 子集 + `sender*` 四键
- ✅ 仓库：**已停止跟踪** `y2m-rs/target*` 构建产物（`.gitignore` 继续忽略）
- ✅ **文档与约定**：根目录 **`agent.md`**（含 tmux / pnpm / 构建测试）；`docs/product/当前实现说明.md` 与本进度表交叉引用（见《当前实现说明》§1.1、§3.11）

### 5.3 管理端 `y2-manage`（Angular，下一阶段主线）

**详细搭建清单（子任务、验收、横向项）**：见 **[`y2-manage-框架搭建任务表.md`](./y2-manage-框架搭建任务表.md)**。

目录：`frontend-monorepo/apps/y2-manage`；包管理 **pnpm**（与根目录 **`agent.md`** 一致）。里程碑与任务编号见 [`任务清单-v2.md`](./任务清单-v2.md) **模块七（MW-*）**、**M3（管理端可用）**。

**现状（截至写入日）**

- ✅ 可 **`pnpm --filter y2-manage build`**（或根脚本 **`pnpm run build:manage`**）通过构建；已具备 **`/login` + 受保护壳路由**、`authGuard` / `loginShellGuard`、`provideCore` + `Y2HttpClient`（Bearer、**401→refresh 单飞**、MFA 路径排除）、**Mock 两步 MFA**（`superadmin` / `groupadmin` + TOTP `123456`）、**临期静默 refresh**（`SessionRenewScheduler` + `refreshSkewSeconds`）。细节与手测说明见 **`apps/y2-manage/README.md`** 与 **[`y2-manage-框架搭建任务表.md`](./y2-manage-框架搭建任务表.md)** §7。
- ⬜ 用户端 Web（y2-chat）已从 monorepo 移除；当前前端主线即本管理端；**对接真实 `auth-service` 的全流程**仍为 AU-04 后手测项。

**后端依赖**

- 登录、会话、用户/角色/设备/审核/审计等页面依赖规划中的 **`auth-service`**（任务清单 **AU-***）。**尚无可用后端时**：可先搭 **路由壳 + UI 骨架**，用 **Mock / 可切换 `environment.apiBaseUrl`** 对齐 [`docs/auth/统一认证中心API定义-v1.md`](../auth/统一认证中心API定义-v1.md) 契约，避免阻塞布局与表单流程。

**建议推进顺序**

1. **壳**：全局布局（顶栏/侧栏或等价）、`<router-outlet>`；路由划分 **`/login`** 与受保护业务区；**路由守卫**占位（未登录跳转登录）。
2. **MW-01**：登录页（用户名密码 + **super_admin 强制 MFA**，与规格一致）。
3. **MW-02**：Access Token **静默刷新**、登出、守卫与 401 统一处理。
4. **MW-03～MW-08**：用户列表/创建禁用、角色权限矩阵、设备管理、CLI 首登审核队列、审计检索与导出（宜 **先只读列表再补写操作**；高危操作二次确认见需求规格）。
5. **MW-09～MW-10**（M7）：Agent 看板与任务树视图，排在 M3 之后或与编排器进度协调。
