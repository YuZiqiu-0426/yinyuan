# 会话与状态机规范 v1

> **适用范围**：本文描述 **统一认证中心（`auth-service`）与 `y2m-server` 接入完成后**，Web/CLI 会话在鉴权与路由侧应遵循的**状态语义与运行时校验目标**。**当前** `y2m-server` **未**在每次业务或广播前执行下文所述会话状态机；实现事实以 [`当前实现说明.md`](./当前实现说明.md) 与仓库根 [`agent.md`](../agent.md) 为准。

## 1. 会话状态

- `active`：允许按权限收发
- `suspended_readonly`：仅接收，不允许发送/管理操作
- `revoked`：会话失效，服务端拒绝并断开连接

## 2. 触发条件

- `active -> suspended_readonly`
  - Web Refresh Token 失效（非高风险）
- `active -> revoked`
  - token 重放
  - 管理员强制下线
  - 异地异常高风险登录
- `suspended_readonly -> active`
  - 重新登录并创建新会话（旧会话不恢复）
- `suspended_readonly -> revoked`
  - 管理员吊销或进一步风险判定

## 3. 运行时校验规则

- 每次业务操作前校验 `session_state`
- 每次广播分发前校验 `session_state + *.recv`
- `revoked` 必须立即断开 WS 连接

## 4. 多端并发规则

- 用户可多会话并存（Web/CLI）
- 状态变更以会话维度为主
- 管理员可按用户维度批量 `revoke`

## 5. 一致性要求

- 会话状态在全节点生效 SLA：`<=5s`
- 状态变更事件必须写入审计日志
- 任何状态机违规转移返回 `AUTH_SESSION_STATE_INVALID`
