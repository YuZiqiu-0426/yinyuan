# y2m-rs 编译部署文档

## 1. 文档目的

本文档用于说明 `y2m-rs` 的编译、打包、部署和基础验证流程，适合开发环境本地运行，也适合最小可用的测试部署。

项目主目录：

```text
y2m-rs/
```

当前项目包含两个主要可执行程序：

1. `y2m-server`：服务端
2. `y2m`：客户端 CLI

## 2. 环境要求

建议准备以下环境：

1. Rust stable 工具链
2. Cargo
3. 可访问目标监听端口的网络环境

可先确认工具链是否正常：

```powershell
rustc --version
cargo --version
```

## 3. 项目结构

当前 workspace 主要结构如下：

```text
y2m-rs/
  Cargo.toml
  src/
    main.rs
  crates/
    common/
    client-core/
    server/
```

其中：

1. `y2m` 位于根包，提供 `init / run / send / chat`
2. `y2m-server` 位于 `crates/server`

## 4. 获取与进入项目

进入项目目录：

```powershell
cd E:\project\YinYuan\document\y2m-rs
```

## 5. 开发环境编译

编译整个 workspace：

```powershell
cargo build
```

只编译客户端：

```powershell
cargo build -p y2m
```

只编译服务端：

```powershell
cargo build -p y2m-server
```

编译完成后，产物默认在：

```text
target/debug/y2m.exe
target/debug/y2m-server.exe
```

在 Linux/macOS 上对应为：

```text
target/debug/y2m
target/debug/y2m-server
```

## 6. 生产或测试发布编译

建议发布时使用 `release`：

```powershell
cargo build --release -p y2m
cargo build --release -p y2m-server
```

发布产物默认在：

```text
target/release/y2m.exe
target/release/y2m-server.exe
```

## 7. 服务端部署

### 7.1 默认行为

`y2m-server` 当前没有额外的命令行参数，启动时会直接监听地址。

默认监听地址：

```text
127.0.0.1:8080
```

可通过环境变量覆盖：

```text
Y2M_SERVER_ADDR
```

### 7.2 本机直接启动

默认地址启动：

```powershell
.\target\release\y2m-server.exe
```

指定监听地址启动：

```powershell
$env:Y2M_SERVER_ADDR="0.0.0.0:8080"
.\target\release\y2m-server.exe
```

Linux/macOS：

```bash
export Y2M_SERVER_ADDR=0.0.0.0:8080
./target/release/y2m-server
```

### 7.3 建议部署方式

最小部署建议：

1. 服务器上只放 `y2m-server`
2. 使用 `release` 版本
3. 明确开放监听端口
4. 使用守护进程或服务管理工具托管

Windows 可先用计划任务或 NSSM 这类工具托管；Linux 可用 `systemd`。

### 7.4 Linux systemd 示例

可执行文件假设放在：

```text
/opt/y2m/y2m-server
```

示例服务文件：

```ini
[Unit]
Description=Y2M Server
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/y2m
Environment=Y2M_SERVER_ADDR=0.0.0.0:8080
ExecStart=/opt/y2m/y2m-server
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启用示例：

```bash
sudo systemctl daemon-reload
sudo systemctl enable y2m-server
sudo systemctl start y2m-server
sudo systemctl status y2m-server
```

## 8. 客户端部署

客户端可以直接分发单个可执行文件 `y2m`。

最小部署材料：

1. `y2m` 可执行文件
2. 一个客户端配置文件

客户端默认配置文件路径：

Windows：

```text
%APPDATA%\y2m\config.json
```

Linux/macOS：

```text
~/.config/y2m/config.json
```

## 9. 客户端初始化

首次建议先执行初始化：

```powershell
.\target\release\y2m.exe init `
  --server-url ws://127.0.0.1:8080/ws `
  --group demo `
  --client alice `
  --heartbeat-interval 10 `
  --download-dir .\downloads
```

常用参数：

1. `--server-url`：服务端 WebSocket 地址
2. `--group`：默认分组
3. `--client`：客户端名称
4. `--token`：预留认证字段
5. `--heartbeat-interval`：心跳间隔覆盖
6. `--download-dir`：文件下载目录

## 10. 配置文件示例

初始化后会生成类似如下配置：

```json
{
  "serverUrl": "ws://127.0.0.1:8080/ws",
  "groupName": "demo",
  "clientName": "alice",
  "token": null,
  "heartbeatIntervalOverrideSec": 10,
  "downloadDir": "downloads"
}
```

## 11. 基础启动方式

### 11.1 控制台接收模式

```powershell
.\target\release\y2m.exe run
```

支持自动重连：

```powershell
.\target\release\y2m.exe run --reconnect-interval-sec 5
```

### 11.2 交互会话模式

```powershell
.\target\release\y2m.exe chat
```

### 11.3 单次发送模式

文本：

```powershell
.\target\release\y2m.exe send text --to bob "hello"
```

JSON：

```powershell
.\target\release\y2m.exe send json --to bob "{\"type\":\"ping\"}"
```

命令：

```powershell
.\target\release\y2m.exe send command --to bob "dir" --timeout 30
```

文件：

```powershell
.\target\release\y2m.exe send file --to bob ".\demo.txt" --timeout 30
```

## 12. 部署后验证建议

建议至少完成以下验证：

1. 服务端启动后可正常监听端口
2. 两个客户端可以成功 `init`
3. 文本单播可达
4. JSON 单播可达
5. 命令请求可返回 `command_result`
6. 文件发送、接收、拒绝、中止可正常工作

## 14. 当前部署限制

当前已知限制如下：

1. 服务端当前主要通过 `Y2M_SERVER_ADDR` 配置监听地址
2. 文件传输断线后当前按失败处理，需要重新发送
3. 自动接受文件、chunk 大小、文件超时等更完整配置项尚未完全开放
4. 服务端运维能力目前是最小可用版，日志、指标、异常清理仍可继续增强

## 15. 推荐发布清单

如果做一套最小可用发布，建议准备：

1. 服务端二进制：`y2m-server`
2. 客户端二进制：`y2m`
3. 一份客户端初始化说明
4. 一份默认配置模板
5. 一份端口开放说明
