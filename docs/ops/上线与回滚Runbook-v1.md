# 上线与回滚 Runbook v1

## 1. 上线前检查

- 数据库迁移脚本已在 staging 验证
- 关键安全用例（重放、CSRF、越权、revoked 断链）通过
- 监控与告警规则已生效
- 管理员 MFA 已开启

## 2. 上线步骤

1. 发布 `auth-service`（灰度 10%）
2. 验证登录/刷新/CLI 审核链路
3. 发布 `chat-service` 鉴权中间件
4. 发布 `user-web` 与 `admin-web`
5. 灰度扩大到 50% / 100%

## 3. 监控指标

- 登录成功率
- token 刷新失败率
- `AUTH_PERMISSION_DENIED` 比率
- `AUTH_RISK_REVOKED` 次数
- WebSocket 断连率
- 权限变更生效延迟（目标 <= 5s）

## 4. 回滚触发条件

- 登录成功率低于基线 20% 以上
- 高风险错误码异常飙升
- 会话状态同步延迟持续超 SLA

## 5. 回滚步骤

1. 暂停灰度并冻结新发布
2. 回滚服务到上一个稳定版本
3. 执行兼容性回滚脚本（仅非破坏性）
4. 验证核心链路恢复
5. 发布事故复盘与修复计划

## 6. 应急联系人机制

- 值班后端负责人
- 安全负责人
- 运维负责人
- 产品/业务值班人

要求：P1 故障 10 分钟内拉群，30 分钟内给出初步结论。
