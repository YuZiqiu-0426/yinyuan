# 权限矩阵与默认角色模板 v1

## 1. 原子权限

- `text.send` / `text.recv`
- `json.send` / `json.recv`
- `command.send` / `command.recv`
- `file.send` / `file.recv`

## 2. 默认角色模板

### `super_admin`

- 全量 `send/recv` 权限
- 可跨组用户与权限管理
- 可审核所有 CLI 首登申请

### `group_admin`

- 组内建议默认：`text.send`、`text.recv`、`json.recv`、`file.recv`
- 不允许跨组创建用户
- 可审核所属组 CLI 首登申请

### `user`

- 默认仅：`text.send`、`text.recv`
- 其他权限按组按人显式授予

## 3. 管理动作权限（非通信）

- `admin.user.create`
- `admin.user.disable`
- `admin.role.assign`
- `admin.permission.update`
- `admin.cli_review.approve`
- `admin.audit.read`

建议：
- `super_admin`：全量管理动作
- `group_admin`：仅组内 `admin.user.create`、`admin.role.assign`、`admin.cli_review.approve`、`admin.audit.read`
- `user`：无管理动作

## 4. 生效优先级

1. 会话状态（`revoked`/`suspended_readonly`）优先
2. 角色管理权限范围（全局/组内）
3. 原子权限 allow/deny
4. 默认拒绝

## 5. 变更治理

- 所有权限变更必须记审计
- 变更后全节点生效 SLA：`<=5s`
- 支持一键回退到“默认角色模板”
