# 配置与密钥管理规范 v1

## 1. 配置分层

- `dev`：本地开发
- `staging`：预发布验证
- `prod`：生产

严禁跨环境复用密钥。

## 2. 必要环境变量

- `AUTH_DB_DSN`
- `AUTH_REDIS_URL`
- `AUTH_JWT_PRIVATE_KEY`
- `AUTH_JWT_PUBLIC_KEYS`
- `AUTH_ACCESS_TOKEN_TTL_SECONDS`
- `AUTH_REFRESH_TOKEN_TTL_SECONDS`
- `AUTH_CSRF_SECRET`
- `AUTH_RATE_LIMIT_WINDOW_SECONDS`
- `AUTH_RATE_LIMIT_MAX_ATTEMPTS`

## 3. 密钥策略

- JWT 使用非对称密钥（RS256/EdDSA）
- 私钥仅保存在密钥管理系统
- 公钥可多版本并存，支持平滑轮换
- 轮换频率：生产建议 90 天

## 4. 轮换流程

1. 生成新密钥对
2. 发布新公钥到校验端
3. 签发切换到新私钥
4. 观察旧 token 自然过期
5. 下线旧公钥

## 5. 安全基线

- 所有配置走环境变量或密钥服务，不入仓
- CI 扫描提交内容中的密钥泄露
- 生产环境禁止调试日志输出敏感字段
