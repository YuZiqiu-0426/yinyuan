# 统一认证中心攻击路径演练与验收清单 v1

## 1. 目的

本清单用于在开发联调和上线前，验证统一认证中心在关键攻击路径上的防护有效性。

覆盖范围：

- Web 认证与会话
- CLI 设备认证与审核流
- 聊天服务实时鉴权
- 管理端高危操作与审计

## 2. 演练前置条件

- `auth-service`、`chat-service`、`user-web`、`admin-web` 已联通。
- 管理端已启用 MFA，且至少有一个 `super_admin` 账号。
- 已配置基础 CIDR 白名单策略。
- 审计日志可检索（按时间、用户、IP）。

## 3. 攻击路径清单（10 条）

### Case 01：Web 密码暴力破解

- 攻击路径：同一用户名短时间高频尝试密码。
- 预期防护：触发登录限流和临时锁定，返回统一失败信息，不泄露账号是否存在。
- 验证步骤：
  1. 对同一账号连续错误登录 10 次。
  2. 观察是否进入锁定窗口。
  3. 检查审计日志与告警事件。
- 通过标准：
  - 错误次数达到阈值后拒绝登录。
  - 响应不泄露用户名存在性。
  - 产生审计与告警记录。

### Case 02：Refresh Token 重放攻击

- 攻击路径：重复使用已轮换失效的 refresh token 刷新会话。
- 预期防护：判定重放，高风险会话直接 `revoked`，连接断开。
- 验证步骤：
  1. 正常登录并刷新一次，拿到新 refresh token。
  2. 用旧 refresh token 再调用刷新接口。
  3. 检查会话状态变化与连接行为。
- 通过标准：
  - 返回 `AUTH_REFRESH_EXPIRED` 或 `AUTH_RISK_REVOKED`。
  - 原会话进入 `revoked`。
  - 审计日志记录重放来源 IP 与会话 ID。

### Case 03：Web 刷新 CSRF 触发

- 攻击路径：第三方站点发起跨站请求触发 `/auth/web/refresh`。
- 预期防护：无有效 CSRF Token 或 Origin 不匹配时拒绝。
- 验证步骤：
  1. 伪造跨域表单或脚本请求刷新接口。
  2. 不带 `X-CSRF-Token` 或带伪造 token。
  3. 检查响应和审计。
- 通过标准：
  - 返回 `AUTH_CSRF_INVALID`。
  - 不签发新 access token。
  - 写入审计日志。

### Case 04：CLI 设备因子伪造（MAC/OS 用户）

- 攻击路径：伪造 `mac` 与 `osLoginUser` 字段冒充可信设备。
- 预期防护：签名校验失败，拒绝登录。
- 验证步骤：
  1. 构造伪造设备字段和无效 `deviceSignature`。
  2. 调用 `/auth/cli/login`。
  3. 检查返回码与设备状态。
- 通过标准：
  - 返回 `AUTH_DEVICE_UNTRUSTED`。
  - 不创建会话。
  - 记录异常设备审计。

### Case 05：CLI 首次登录绕过审核

- 攻击路径：首次登录后直接尝试拿 token，跳过审核流程。
- 预期防护：返回 `AUTH_CLI_REVIEW_PENDING`，不可进入 active 会话。
- 验证步骤：
  1. 用新设备首次登录。
  2. 在审核前反复请求访问 chat 接口。
  3. 检查授权结果。
- 通过标准：
  - 未审核状态无法拿到可用会话 token。
  - 所有业务操作被拒绝。
  - 审计日志包含 reviewId。

### Case 06：group_admin 越权创建跨组用户

- 攻击路径：`group_admin` 指定非本组 `groupName` 创建用户。
- 预期防护：拒绝并返回范围越权错误。
- 验证步骤：
  1. 用 `group_admin` 调用 `POST /admin/users`。
  2. 请求体中填入其他组名。
  3. 检查响应与数据库变更。
- 通过标准：
  - 返回 `AUTH_GROUP_SCOPE_DENIED`。
  - 无用户创建记录。
  - 审计记录越权尝试。

### Case 07：只读挂起会话发送消息

- 攻击路径：会话已 `suspended_readonly`，继续调用 `POST /chat/text/send`。
- 预期防护：禁止发送，返回只读错误码。
- 验证步骤：
  1. 触发 refresh 失效使会话降级为只读。
  2. 使用原 access token 发文本消息。
  3. 检查消息是否入路由。
- 通过标准：
  - 返回 `AUTH_SESSION_READONLY`。
  - 消息不入库、不广播。
  - 记录拒绝审计。

### Case 08：revoked 会话继续监听广播

- 攻击路径：会话被管理员强制下线后仍尝试维持 WS 连接监听。
- 预期防护：服务端立即断连，无法继续接收广播。
- 验证步骤：
  1. 建立 WebSocket 连接并保持监听。
  2. 管理端将该会话标记 `revoked`。
  3. 观察连接状态和后续推送。
- 通过标准：
  - 连接在生效窗口内断开。
  - 不再接收任何消息。
  - 审计含强制下线动作记录。

### Case 09：CIDR 白名单绕过

- 攻击路径：来源 IP 不在白名单网段，尝试登录或调用受限接口。
- 预期防护：拒绝连接与操作。
- 验证步骤：
  1. 配置仅允许 `10.0.0.0/24`。
  2. 用 `10.0.1.x` 地址发起登录或请求。
  3. 检查返回与日志。
- 通过标准：
  - 返回 `AUTH_IP_MISMATCH` 或等价拒绝码。
  - 不创建会话。
  - 审计包含匹配失败规则信息。

### Case 10：权限变更延迟窗口利用

- 攻击路径：管理员移除某用户 `text.send` 后，用户立即尝试发送消息。
- 预期防护：在 SLA 内生效，超时触发告警。
- 验证步骤：
  1. 用户保持在线状态。
  2. 管理端移除 `text.send`。
  3. 在 0~10 秒内重复发送消息并记录结果时间点。
- 通过标准：
  - 在 `<=5s` 内开始稳定拒绝发送。
  - 若超过 SLA，触发运维告警。
  - 审计记录权限变更与拒绝操作链路可关联。

## 4. 回归测试分层建议

- 冒烟层（每次发布必跑）：Case 02 / 03 / 05 / 07 / 08
- 日常层（每日构建）：Case 01 / 04 / 06 / 09
- 变更层（权限与会话改动后）：Case 10

## 5. 验收输出模板

每条 Case 建议记录：

- 用例编号
- 执行环境（dev/staging/prod）
- 执行人
- 执行时间
- 结果（pass/fail）
- 证据链接（日志、截图、请求响应）
- 失败原因与修复单号

## 6. 结论判定规则

- 高风险用例（Case 02 / 03 / 04 / 08）必须全部通过。
- 任一高风险失败，版本不可发布。
- 中低风险失败可在有临时缓解方案且审批通过后灰度发布。
